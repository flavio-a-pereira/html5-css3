CAPÍTULO 1 – EVOLUÇÃO DA INTERNET

A internet surgiu para suprir necessidades durante a guerra fria(1949). Para tentar se igualar aos avanços tecnológicos da URSS os EUA criaram a DARPA que era uma agencia voltada para estudos tecnológicos, com fins militares. Uma das tecnologias criadas nesta agencia foi a ARPANET, feita para unir bases militares no território americano e proteger seus dados. Ela uniu 4 pontos: um SDS Sigma 7 na UCLA (CA), um SDS 90 na Universidade de Stanford(CA), um IBM 370/75 no Centro de Matemática de Los Angeles(CA) e uma maquina militar DEC PDP-10 na Base Militar de Utah.

Como as maquinas tinham linguagens diferentes, foi criado um protocolo que elas pudessem interagir(net control protocol- NCP). Em 1972 já ligava pontos em todos os EUA. Porem o NCP tinha a limitação de parar para que 2 máquinas interagissem, e já não conseguia dar agilidade para estas ligações. Daí surgiu o TCP, porém apresentava problemas de identificação dos pontos. Para resolver este problema foi criado o IP para identificação dos pontos. Surgia aí o TCP/IP. Como a rede aumentou muito os militares dividiram a ARPANET em 3 redes MILNET, NSFNET e comerciais, e passaram a controlar só a MILNET.

As redes quiseram se manter conectadas, fazer uma interconnect networking, que virou internetworking e simplesmente internet.

A rede de cabos que liga a internet pode ser vista abaixo e foram retiradas do site submarine cablemap. Podemos obter mais informações sobre esta rede no video do youtube "How does the Internet Work".

Em 1993 foi criado um novo protocolo chamado hipertex protocol (HTTP)e a linguagem HTML (hiper text markup language)e o termo world wide web(www). O primeiro navegador criado para acessar HTTP foi o mosaic. A WWW é uma rede específica dentro da internet para englobar servidores HTTP.

CAPITULO2 – COMO FUNCIONA A INTERNET

Os computadores, celulares, tablets,etc são equipamentos eletrônicos e funcionam através de sinais elétricos em forma de uma onda quadrada. Estes sinais são apresentados com os algarismos 0 e 1 (sem sinal 0 e com sinal 1). Eles são conhecidos com dígitos binários(binary digit) ou bit. A porção mínima para se apresentar algum dado em computação precisa de 8 bits. Esta sequencia é chamada de 1 byte.

A conexão com a internet

Quando falamos em conexão com a internet, os computadores, celulares e demais aparelhos que pretendem se conectar são chamados de clientes. Tem esse nome por que querem acessar um serviço. Para fazermos este acesso precisamos de um aparelho chamado de modem. Este aparelho serve para converter o tipo de onda que sai dos computadores, celulares, etc (onda quadrada) para serem entendidos pela linha de telefone, antena de celular etc, assim como converter o sinal que vem da internet(onda senoidal) para o formato entendido pelos aparelhos clientes.

Transformar a onda quadrada para senoidal chama-se modulação (mod) e o inverso demodulação (dem).

Os conteúdos da internet estão armazenados em servidores específicos e ambos são identificados por um número chamado IP. O local que guarda a relação entre os números de IP e seus nomes é chamado de servidor DNS(sistema de nome de domínio). Quando digitamos um nome de um site para acesso, primeiro ele o procura neste servidor DNS, retorna o número do IP para o modem, que manda esta mensagem para a internet, que retorna a informação pedida.

Situação 1 : Dados guardados no servidor

Qualquer cliente digita o nome do site, que vai busca-lo no servidor DNS, que retorna a informação do seu numero de IP , que por sua vez encaminha para o servidor de hospedagem, que retorna o conteúdo pedido para o cliente. O servidor de hospedagem é S e os clientes são A,B,C.

Situação 2: Guardar os dados no PC

Dados guardados no PC só podem ser consultados pelo próprio usuário do computador, por que não possui um domínio cadastrado em um servidor DNS. Além disso, para consulta publica, é necessário que o site esteja arquivado em um servidor apropriado, chamado de servidor de hospedagem.

Para acessar um site, digitamos uma URL (localizador de recurso único), por exemplo WWW.github.com/gustavoguanabara. O domínio é a 1ª parte, em github.com., o .com é chamado de TLD (domínio em alto nível) ou GTLD pois tem caráter genérico como .net, .io, .edu, .store, .gov. Existe também os domínios do tipo CCTLD (com códigos de países) como .br, .us,.uk,.ar. O WWW é chamado de sub-domínio. O que vem depois da / é o caminho.

  • Exemplo : gustavoguanabara.github.io = URL
  • gustavoguanabara=sub-dominio
  • githib.io=domínio
  • .io=gtld

CAPÍTULO 3 – COMO FUNCIONA HTML E CSS

HTML é uma linguagem de desenvolvimento (não é de programação) para criar sites. Significa Hypertext Markup Language. O hipertexto é uma área em que se pode clicar encima e ser enviado para outro conteúdo. Markup significa que é uma linguagem definida em marcas, não possuem variáveis, condições,vetores, objetos, etc.

CSS são folhas de estilo em cascata (Cascading Style Sheets). Acrescenta características em cascata.

A linguagem HTML é focada em conteúdo como textos, imagens, vídeos, tabelas. Porém para tornar a informação mais agradável de ser consultada, precisamos usar atrativos visuais para reter os visitantes. Estes atrativos são as CSS. Por exemplo posição,tamanho,sombras e cor de imagens, textos sobrepostos sobre imagens .

P s: as interações de um site são feitos em javascript(esta é linguagem de programação).
